### 승인 큐 API

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/approvals` | 대기 중인 제안 목록 |
| POST | `/api/approvals/:id/approve` | 승인 → css_selector 교체 + status=healthy + healed_count+1 |
| POST | `/api/approvals/:id/reject` | 거부 → crawler status=failed |

### tryHeal() 3분기 로직

```
Python /heal 응답
    ├─ status='healed' AND confidence ≥ threshold/100
    │    → 자동 복구 (css_selector 교체, status=healthy)
    │
    ├─ status='healed' AND confidence < threshold/100
    │    → 승인 큐 저장 (heal_proposals insert, status=pending)
    │
    └─ status='failed' | 'no_change_needed' | 기타
         → 크롤러 status=failed, 제안 저장 안 함
```

## 승인 큐 버그 수정

### 버그 1 — 빈 제안이 큐에 저장되는 문제

**증상**: 승인 큐에 `(제안 없음)` 항목이 쌓임  
**원인**: `crawler.js` `tryHeal()`의 `else` 분기가 `status='failed'`인 경우에도 `db.proposals.insert()`를 호출  
**수정**: 3분기 조건으로 분리 — `status === 'healed'`일 때만 제안 저장, 그 외는 저장 없이 `status=failed`

```js
// Before (버그): 단순 if/else → failed여도 insert
// After (수정): 명시적 3분기
if (result.status === 'healed' && confidence >= thresholdRatio) {
  // 자동 복구
} else if (result.status === 'healed' && confidence < thresholdRatio) {
  db.proposals.insert(...);   // 신뢰도 미달만 저장
} else {
  // status=failed → 저장 안 함
}
```

### 버그 2 — Tailwind 셀렉터가 BeautifulSoup에서 실패

**증상**: ML 서버가 항상 `status: 'failed'` 반환, 자가치유 불가  
**원인**: Playwright가 생성한 CSS 셀렉터에 Tailwind 임의값 클래스(`min-h-\[170px\]`, `w-\[516px\]`) 포함 → BeautifulSoup `select_one()` 파싱 오류  
**수정**: `capstone_dataset/inference/healer.py`에 `_safe_select_one()` 헬퍼 추가

```python
def _safe_select_one(soup, selector):
    try:
        node = soup.select_one(selector)
        if node: return node
    except Exception:
        pass
    # Tailwind arbitrary values 제거 후 재시도
    simplified = re.sub(r'[\w-]+\\\[.*?\\\]', '', selector)
    simplified = re.sub(r'\.{2,}', '.', simplified).strip()
    if simplified and simplified != selector:
        try:
            return soup.select_one(simplified)
        except Exception:
            pass
    return None
```

## 셀렉터 선택기 팝업 제거 도구

**배경**: 크롤러 등록 Step 3(요소 선택) 화면에서 사이트 팝업·모달이 뜨면 원하는 요소를 클릭할 수 없는 문제

### 변경 파일

| 파일 | 내용 |
|------|------|
| `server/server.js` | WebSocket 핸들러 3개 추가 |
| `client/screens.jsx` | WizardStep3 팝업 제거 툴바 + `removeMode` 상태 |

### 서버 — 새 WebSocket 메시지 타입

| type | 동작 |
|------|------|
| `keypress` | `page.keyboard.press(msg.key)` — ESC 전송에 사용 |
| `remove_overlays` | JS 주입으로 `[role="dialog"]`, 고 z-index fixed/absolute 요소, 공통 팝업 클래스명 일괄 삭제 + body overflow 해제 |
| `remove_element` | `document.elementFromPoint(x, y).remove()` — 좌표 기준 단일 요소 삭제 |

### 클라이언트 — 팝업 제거 툴바

URL 바 아래, 캔버스 위에 항상 노출되는 툴바:

```
팝업 제거:  [ESC]  [자동 제거]  [요소 지우기]
```

- **ESC**: Escape 키 전송 → 대부분의 네이티브 모달/오버레이 닫기
- **자동 제거**: `remove_overlays` 전송 → 감지된 팝업 전부 일괄 삭제
- **요소 지우기 (토글)**: ON 시 클릭이 CSS 셀렉터 선택 대신 해당 요소 DOM 삭제로 동작, 커서 `not-allowed`로 변경 + 경고 레이블 표시

## ML 서버 실행 오류 진단 및 수정

**증상**:
```
ERROR: Error loading ASGI app. Attribute "app" not found in module "main".
[healer] 오류: fetch failed
```

**원인**: `capstone_dataset/` 루트에 있는 `main.py`는 ML 파이프라인 스크립트로 FastAPI `app` 객체 없음.  
실제 API 서버는 `capstone_dataset/api/main.py`에 있음.

**올바른 실행 명령**:
```bash
# capstone_dataset/ 에서
cd capstone_dataset
uvicorn api.main:app --reload --port 8000   # ← 정확한 명령
```

**잘못된 명령 (오류 발생)**:
```bash
uvicorn main:app --reload --port 8000       # ← main.py를 잘못 가리킴
```

## 최종 서버 실행 체크리스트

```bash
# 1. Node.js 서버 (Express + Playwright + SQLite)
cd /path/to/Playground
npm start                          # → http://localhost:3001/Mender.html

# 2. Python ML 서버 (FastAPI + Random Forest + GPT-4o-mini)
cd /path/to/capstone_dataset
source venv/bin/activate
uvicorn api.main:app --reload --port 8000

# 3. 동작 확인
curl http://localhost:3001/api/stats
curl http://localhost:3001/api/approvals
```

> Python 서버 없이 Node.js만 실행해도 크롤링 및 UI는 정상 동작.  
> 자가치유 호출 시 `[healer] 오류: fetch failed`가 발생하며 크롤러 상태는 `failed`로 표시됨.
